### Test files
| File | Tests |
|------|-------|

| `e2e/tests/04-checkout.spec.ts` | Guest redirect, Logged-in checkout, Form validation |
| `e2e/tests/05-admin.spec.ts` | Admin order status update |

### Root cause of most failures
The `CartDrawer` component lives in the layout and is **always in the DOM** (hidden via `translate-x-full`). This means every price, product name, and action button appears twice ‚Äî once in main content, once in the off-screen drawer. Playwright's strict mode throws when a locator matches multiple elements.

### Fixes applied
| Test | Problem | Fix |
|------|---------|-----|
| Logout | `waitForURL(/^\/(auth\/login)?$/)` ‚Äî `^` anchors to start of full URL, not path | Removed `^`: `/\/(auth\/login)?$/` |
| Product detail | `getByRole('heading', { name: 'Size' })` partially matched h1 "Oversized **S**ize" | Added `{ exact: true }` |
| Add item | `getByText('Oversized Cotton Tee')` matched h1 + drawer `<p>` | Added `.first()` |
| Update quantity | `getByRole('button', { name: '+' })` matched main + drawer | Scoped to `page.getByRole('main')` |
| Remove item | `getByText('Your cart is empty.')` matched main + drawer | Scoped to `page.getByRole('main')` |
| Logout (navbar) | `getByRole('link', { name: 'Sign in' })` ‚Äî Navbar renders `<a><button>Sign in</button></a>` (invalid HTML nesting), Playwright doesn't find the `<a>` as a "link" | Replaced with `/account` redirect check |

## 2. Git & GitHub Setup

### Problem
- `/usr/bin/git` blocked by Xcode license stub ‚Üí use `/Applications/Xcode.app/Contents/Developer/usr/bin/git`
- `gh` CLI not installed, `brew` not installed

### Solution
Installed `gh` via webi:
```bash
curl -sS https://webi.sh/gh | sh
~/.local/bin/gh auth login  # device flow at github.com/login/device
```

Push with token:
```bash
TOKEN=$(~/.local/bin/gh auth token)
git remote set-url origin "https://jpayanshi:${TOKEN}@github.com/jpayanshi/sample-project.git"
git push
```

## 3. Admin Pages ‚Äî Auth Guard

### Problem
`/admin/products` and `/admin/orders` were accessible to any URL visitor ‚Äî no frontend auth check.

### Fix
Added three-stage guard to both pages:
```tsx
if (authLoading) return <div className="py-20 text-center">Loading‚Ä¶</div>;
if (!authData?.user) { router.push('/auth/login'); return null; }
if (authData.user.role !== 'ADMIN') { router.push('/'); return null; }
```

Backend was already protected via `authenticate + adminOnly` middleware.

## 4. Logout Bug ‚Äî Navbar Showing Stale User Name

### Symptom
After clicking "Sign out", the Navbar still showed the logged-in user's name.

### Investigation
1. **Backend verified working** ‚Äî curl test confirmed:
   - `POST /api/auth/logout` ‚Üí `Set-Cookie: token=; Path=/; Expires=Thu, 01 Jan 1970; HttpOnly; SameSite=Lax`
   - Subsequent `GET /api/auth/me` ‚Üí `401 Unauthorized`
2. **clearCookie fix** ‚Äî original `res.clearCookie('token')` called without options; fixed to pass matching `{ httpOnly, secure, sameSite }` attributes so the browser definitively clears the cookie
3. **Real root cause** ‚Äî React Query race condition:
   - `router.push('/')` (soft navigation) keeps the Navbar component mounted
   - `queryClient.clear()` removes cache but immediately triggers background refetches for all still-subscribed queries
   - The `me` background refetch raced with cookie clearing ‚Üí could return user data before React Query state settled

### Fix
Replace `router.push('/')` with `window.location.href = '/'` in the logout `onSuccess`:
```typescript
// Before
onSuccess: () => {
  queryClient.clear();
  queryClient.setQueryData(['me'], null);
  router.push('/');
},

// After
onSuccess: () => {
  window.location.href = '/';
},
```

Full page reload:
- Tears down all React / React Query state
- Browser makes fresh requests with no cookie ‚Üí `me` returns 401 ‚Üí Navbar renders "Sign in" cleanly

## 5. Customer Account Page ‚Äî Tabbed Layout

### Before
Single page with name, email, and a basic order history list.

### After
Two-tab layout at `/account`:

#### Settings tab
- **Profile section** ‚Äî editable name and phone; email read-only; inline Save/Cancel (only appears on change)
- **Saved Addresses section** ‚Äî list all addresses with Remove; expandable "Add address" form

#### Orders tab
- Collapsible order rows: order number, date, status badge, total
- Expanded view: each item with product name, size, color, quantity, line total
- Shipping address shown at bottom of expanded order

### Backend additions
| Endpoint | Description |
|----------|-------------|
| `PUT /api/auth/profile` | Update name and phone |
| `GET /api/addresses` | List user's saved addresses |
| `POST /api/addresses` | Create new address |
| `DELETE /api/addresses/:id` | Delete address (with ownership check) |

### Schema change
Added `phone String?` to the `User` model ‚Üí `prisma db push` applied.
